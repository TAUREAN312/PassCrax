<h1>Contributing to PassCrax</h1>

<p>Thank you for considering contributing to PassCrax! We welcome all contributions that help improve this project. Please follow these guidelines when making changes.</p>

<hr>

<h2>1. How to Contribute</h2>
<ul>
    <li>Fork the repository and create a new branch for your changes.</li>
    <li>Make sure your code follows the project's coding style.</li>
    <li>Test your changes before submitting a pull request.</li>
    <li>Write clear commit messages.</li>
</ul>

<hr>

<h2>2. Code Style</h2>
<ul>
    <li>Use meaningful variable names.</li>
    <li>Keep code clean and readable.</li>
    <li>Follow indentation and spacing conventions.</li>
    <li>Add comments to explain complex logic when necessary.</li>
</ul>

<hr>

<h2>3. Reporting Issues</h2>
<p>If you find a bug or have a feature request, open an issue with:</p>
<ul>
    <li>A clear title and description.</li>
    <li>Steps to reproduce the issue.</li>
    <li>Any relevant error messages or screenshots.</li>
</ul>

<hr>

<h2>4. Submitting a Pull Request (PR)</h2>
<ul>
    <li>Fork the repo and create a feature branch (<code>git checkout -b feature-name</code>).</li>
    <li>Commit your changes with a meaningful message (<code>git commit -m "Added feature"</code>).</li>
    <li>Push your branch and submit a PR.</li>
    <li>Reference relevant issues if your PR fixes one (e.g., <code>Fixes #12</code>).</li>
</ul>

<hr>

<h2>5. Community Guidelines</h2>
<ul>
    <li>Be respectful and collaborative.</li>
    <li>No harassment or offensive behavior.</li>
    <li>Help improve PassCrax by providing constructive feedback.</li>
</ul>

<p><b>Thank you for contributing!</b></p>
