    <h1>PassCrax Contributor Guidelines</h1>
    
    <p>Welcome to PassCrax! We appreciate contributions that help improve this project. Please follow these guidelines when contributing.</p>

    <h2>1. How to Contribute</h2>
    <ul>
        <li>Fork the repository and create a new branch for your changes.</li>
        <li>Write clear and concise commit messages.</li>
        <li>Ensure your code follows the project's coding style.</li>
        <li>Test your changes before submitting a pull request.</li>
    </ul>

    <h2>2. Code Style</h2>
    <p>Keep the code clean and readable:</p>
    <ul>
        <li>Use meaningful variable names.</li>
        <li>Follow indentation and spacing conventions.</li>
        <li>Comment complex logic when necessary.</li>
    </ul>

    <h2>3. Reporting Issues</h2>
    <p>If you find a bug or have a feature request, open an issue with:</p>
    <ul>
        <li>A clear title and description.</li>
        <li>Steps to reproduce the issue.</li>
        <li>Any relevant error messages or screenshots.</li>
    </ul>

    <h2>4. Submitting a Pull Request</h2>
    <p>When submitting a pull request:</p>
    <ul>
        <li>Describe your changes clearly in the PR description.</li>
        <li>Ensure all tests pass before submission.</li>
        <li>Tag relevant issues if your PR fixes one (e.g., <code>Fixes #12</code>).</li>
    </ul>

    <h2>5. Community Guidelines</h2>
    <p>Be respectful and collaborative. Harassment or offensive behavior will not be tolerated.</p>

    <p>Thank you for contributing to PassCrax!</p>
